# AI Advertising Studio

A campaign-management workspace: create campaigns, generate AI ad creatives,
organize assets and audiences, connect supported ad accounts, and (where the
platform API allows) schedule, publish, and report — using only official APIs.

## 1. Architecture
Dashboard module under `/dashboard/ads`. Provider capabilities come from the
`AD_PLATFORMS` registry in `constants.ts` (formats, supportsPublish,
supportsReporting, envKeys, docs). AI copy is generated by `lib/ads/generate.ts`
(Claude when `ANTHROPIC_API_KEY` is set, deterministic placeholder otherwise).
Industry starter templates live in `lib/ads/templates.ts` (+ the DB seed).

## 2. Dashboard pages
Campaign Dashboard, Create Campaign (wizard), Ad Creative Studio, Creative
Library, Connected Ad Accounts, Campaign Calendar, Campaign Reports, Audience
Library, Settings — all under the "AI Advertising Studio" sidebar group.

## 3. Campaign workflow
Wizard: Business → Objective → Platforms → Creative types (filtered to the chosen
platforms' supported formats) → Audience → Review → Create. Creating a campaign is
free (draft); afterward you generate AI creatives (copy pack + A/B variations)
which are saved to the campaign. Status lifecycle: draft → scheduled → publishing
→ running → paused → completed → failed.

## 4. Database (migration `0019_advertising.sql`)
`ad_campaigns`, `ad_creatives`, `ad_assets`, `connected_ad_accounts`,
`campaign_reports`, `campaign_variations`, `campaign_audiences`,
`campaign_history`, `campaign_templates`. Owner-only RLS on user tables;
`campaign_templates` is public-read and seeded with 12 industries. Ad-account
tokens are stored in `encrypted_token` (encrypt at rest) and never returned to
the client.

## 5. API routes
`/api/ads/campaigns` (CRUD), `/api/ads/creatives` (list/archive/delete),
`/api/ads/creatives/generate` (AI copy, credit-charged), `/api/ads/templates`,
`/api/ads/accounts` (list/connect/disconnect), `/api/ads/audiences` (CRUD),
`/api/ads/reports`. All owner-scoped; generation is rate-limited.

## 6. Credit usage
Charged for AI generation only (viewing is free): ad copy 2, image 3, video 80,
analysis 2 (see `AD_CREDIT_COSTS`). Campaign creation is free (a draft). Costs are
shown before generating; insufficient balance returns 402 with an upgrade prompt.
Placeholder (no AI key) is free.

## 7. Security
- Official OAuth per platform; connections activate only when that platform's Ads
  API app credentials (envKeys) are configured and approved.
- Tokens encrypted at rest; never sent to the browser.
- Owner-only RLS; campaign actions logged to `campaign_history`; generation
  rate-limited; users authenticate their own ad accounts.

## 8. Testing
1. Run `0019_advertising.sql`.
2. Create Campaign → wizard → Create → generate copy (charges credits only with
   a real AI key; otherwise placeholder).
3. Creative Library lists/searches/archives/deletes creatives.
4. Connected Ad Accounts shows each platform as "Not enabled" until its env keys
   are set (honest gating).
5. Reports shows "limited" until a reporting-capable account is connected.
6. Templates load (DB or constant fallback).

## 9. Supported platform capabilities
Facebook, Instagram, Google, YouTube, LinkedIn, Pinterest, TikTok — each with its
real supported formats. Publishing + reporting require each platform's approved
Ads API app; the registry makes adding/enabling providers a one-entry change.

## 10. Future roadmap
- Wire each platform's official Ads API (Meta Marketing, Google Ads, TikTok
  Business, LinkedIn Marketing) for real publishing + metrics after app review.
- AI image/video creatives directly into ad_creatives (reuse the media engine).
- Budget pacing + AI recommendations panel; real reporting sync into
  `campaign_reports`; drag-and-drop calendar scheduling.
